🧬 HMM Homologue Finder

A reproducible pipeline that finds distant homologues of a protein family across public phage/viral databases — including genes that standard annotation misses — and returns validated sequences, tables, trees, synteny maps, and a publication-ready package. Give it a seed FASTA; it does the rest.

HMMER profile search six-frame translation ORF validation iterative convergence NT + AA output works for any protein

What it does

It builds a profile HMM from your seed sequences, searches ten public databases (translating genome databases in all six reading frames so it can reach unannotated genes), reconstructs and validates the exact ORF each hit came from, then uses the discovered sequences to re-seed the search and repeat until the family stops growing (convergence). It finishes by clustering the homologues, drawing synteny figures and a phylogenetic tree, and assembling everything into a labelled folder.
It is named for the project it was built on, but it is general: point it at any protein family's seed FASTA. Output files keep the family_ name prefix (cosmetic).

How to run

Option 1 — double-click (easiest)

In Finder, open scripts/ and double-click Run HMM Homologue Finder.command. It checks the software (installing anything missing the first time), then asks for your seed FASTA and runs everything.

Option 2 — terminal, interactive

conda activate hmm-discovery
python3 scripts/hmm_finder.py
#   → it prompts:  seed FASTA >   (drag your file in, press Enter)

Option 3 — terminal, explicit

python3 scripts/hmm_finder.py --fasta my_seeds.faa --iterations 3 --cpu 8

The workflow, step by step

1
Build the HMM. Align the seeds (MAFFT), trim (trimAl), build a profile HMM (hmmbuild), and confirm it recovers its own seeds.
2
Search 10 databases. hmmsearch at E ≤ 1e-5. Genome databases are translated in all six frames (min ORF 30 aa) so unannotated genes are reachable.
3
Extract & validate each hit. Reconstruct the exact ORF from genomic coordinates, delimit the domain by the HMM envelope, and confirm it is a real ORF (no internal stops; sits in a coding locus). Save nucleotide and amino-acid sequence + a 35-column evidence table.
4
Iterate. Deduplicate the hit sequences and use them to re-seed the search; repeat (default 3 rounds). Stable counts = converged.
5
Characterise. CD-HIT clustering, clinker synteny figures, an IQ-TREE phylogeny, MEME/FIMO motifs, GFF3 tracks, and real-sequence GenBank files (named by organism).
6
Package. Assemble a labelled, self-contained output folder with all data, figures, sequences, and a methods write-up.

The scripts (click to expand)

hmm_finder.py — the one-command driver
The entry point. Prompts for the seed FASTA (or takes --fasta), runs the software preflight, then orchestrates every step below and assembles the output package. This is the only script you normally run. 
python3 scripts/hmm_finder.py            # interactive
python3 scripts/hmm_finder.py --fasta seeds.faa --iterations 3
check_tools.py / setup.sh — environment check & install
check_tools.py verifies every required program and Python package and prints a ✓/✗ table. setup.sh creates the hmm-discovery conda environment and installs anything missing (HMMER, MAFFT, trimAl, Prodigal, seqkit, CD-HIT, IQ-TREE, MEME/FIMO, clinker). The pipeline calls the check at startup.
extract_validated_hits.py — the core step
For every hit, reconstructs the ORF from its genomic coordinates (1-based, strand/frame-correct), finds the domain via the HMM envelope, and validates it (internal-stop count, ORF coverage, Prodigal overlap). Writes hits.tsv (with a source_type column), the AA and NT FASTAs, full-ORF FASTAs, and the deduplicated seed set for the next round. Hits from protein databases (SwissProt, RefSeq/INPHARED proteins, Pfam) are also captured — their amino-acid sequence is fetched from NCBI and marked source_type=annotated_protein — so no hits are dropped, whatever the protein.
annotate_organism.py — add phage names
Adds an organism column to the hit table (e.g. “Escherichia phage vB_EcoP_G7C”), fetched from NCBI; uncultured metagenomic genomes are marked “uncultured virus (database)”.
cluster_and_clinker_corrected.py — clusters & synteny
Clusters the validated domains with CD-HIT (40% id, 80% cov) and builds clinker gene-neighbourhood figures grouped by cluster, with the genuine homologue ORF as the central gene.
build_real_genbanks.py — viewable GenBank files
Fetches each hit genome (NCBI efetch / catalogue stream) and writes neighbourhood GenBank files with real sequence + the homologue and flanking genes — open directly in Artemis / Geneious / UGENE. Files are named by phage where the name is known.
build_tree_of_hits.py — phylogeny
Aligns the unique domains (MAFFT), trims (trimAl -gt 0.5), and infers a maximum-likelihood tree (IQ-TREE, ModelFinder, 1000 ultrafast bootstrap).
assemble_publication_v2.py — package builder
Collects all outputs into the labelled, numbered folder layout with a convergence analysis and READMEs.

Why “ORF validation” matters

A profile HMM can light up on a stretch of translated DNA that isn't a real gene. To avoid reporting noise, every hit must (a) come from a clean stop-to-stop ORF with no internal stop codons, and (b) sit inside a genuine coding locus (it overlaps a Prodigal-predicted gene). Interestingly, these homologues are usually antisense or in a different frame to the predicted gene — which is exactly why standard annotation missed them. Each hit's evidence is recorded in the table, so nothing is taken on faith.

Requirements

One-time: the hmm-discovery conda environment (run bash setup.sh). It needs conda/mamba — install Miniforge first if you don't have it. Tools: HMMER 3.4, MAFFT v7.526, trimAl v1.5, Prodigal V2.6.3, seqkit v2.13.0, CD-HIT 4.8.1, IQ-TREE 3.1.2, MEME/FIMO 5.5.9, clinker v0.0.32, plus Biopython & pandas. The search engine (run_all_database_benchmark.py) lives in the HMM-Discovery deployable repo, which the pipeline calls automatically.
Runtime & internet: a full 3-iteration run streams several GB of databases and takes hours — run it and walk away (it keeps the Mac awake and is resumable).
